## Learned User Preferences

- For static campaign HTML, sometimes wants a minimal navbar: logo and primary CTA only, with mid-page nav links removed when requested for a file.
- Requests stronger large-viewport layout and typography (wide breakpoints) on campaign HTML pages when asked.
- For selected designs, wants a glassmorphic navbar with scroll-driven behavior (e.g., different treatment over the hero vs. below it, or hide on scroll down and show on scroll up).
- When refining campaign landings (e.g. `5.html`), often wants an Apple-style minimal, conversion-focused treatment: strong whitespace, reduced clutter, clear hierarchy, and a single dominant primary action.
- When refining the product grid on a campaign page, may explicitly keep **six** product cards and ask to ignore pasted specs that cap the count at three (or four); still apply other refinements (copy, CTAs, spacing).

## Learned Workspace Facts

- Bhakti campaign work lives in numbered static HTML files (e.g., `2.html`, `3.html`, `5.html`, `6.html`) and variants such as `design1.html`.
- For `5.html`, assets in the page folder include `logo.png`, `trust-seal.png` (IndiaMart seal, clickable in nav/footer), and `onestar.png` (One Star emblem beside the seal, not a link); badges belong in the navbar and footer, not the hero.
- Local preview is often via VS Code Live Server at `http://127.0.0.1:5500/` (e.g., `/5.html`).
